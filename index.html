LLM-Powered Multi-Agent Proactive Communication System for Embodied Intelligence

Submitted to LM4Plan @ AAAI 2025

Robots execute decisions based on collected and self-generated information, communicated through the message pool. The message pool stores initial setup information, environmental data, and task-specific data, which robots actively query and update. The manager oversees data management, including writing, deleting, reading, and checking data, ensuring data integrity and accessibility for effective decision-making and task execution.

Abstract

Multi-robot collaboration through large language models (LLMs) faces significant challenges in information sharing and coordinated decision-making, particularly as the number of robots increases. To address these limitations, we develop a novel framework that enables proactive information seeking through a centralized message pool architecture, where robots actively retrieve and utilize information rather than passively relay data.

OWe focus on complex coordination tasks that require multiple robots to maintain consistent shared context while making independent decisions. Our key innovation, a hierarchical access protocol with code-based queries, achieves significantly better information retention than dialogue-based approaches (30% vs 55% forgetting rate after 20 rounds). Experiments in both simulation and physical environments demonstrate that our framework achieves 24% faster task completion and 32% lower error rates compared to baseline systems, while successfully generalizing across varying task complexities involving 2-5 robots.

Message Pool Communication

Our framework introduces a novel message pool architecture that facilitates efficient multi-robot communication and information sharing. The message pool serves as a centralized information hub with two main components:

Set-up Information

The set-up information section stores essential static data including:

  • Task Info: Detailed specifications of tasks to be performed
  • Environmental Info: Data about the operational environment
  • Initial Info: Starting configurations and parameters

Updating Information

The current robots information section maintains dynamic data that reflects the ongoing state of operations:

  • Active Querying: Robots proactively retrieve relevant information using code-based queries
  • Data Management: The manager component oversees data operations including writing, deleting, reading, and verification
  • Real-time Updates: Continuous updates of robot states and task progress

This architecture enables robots to maintain a consistent shared context while making independent decisions, significantly reducing information loss compared to traditional dialogue-based approaches.
